34 ## Structural data - apo structure and sdf for each hit in a named target
35 ### Using the API
36 
37 **1. Getting target data**
38 
39 First, init an instance of GetTargetsData from fragalysis_preproc/data.py:
40 
41 ```
42 from data import GetTargetsData
43 
44 search = GetTargetsData()
45 ```
46 
47 Next, we need to set a target name that we want to search for. This is done by setting the target_name_url attribute:
48 
49 ```
50 # e.g. set NUDT5A as the target for the search
51 search.set_target_name_url(target='NUDT5A')
52 ```
53 
54 Now, we can get a json response, which gives back the info specified by the fragalysis RESTful api (see <insert documentation link when it exists>):
55 
56 ```
57 # get the json response
58 results = search.get_target_json()
59 ```
60 
61 Finally, we can parse the json with a built-in function to return id's (db id) for the entries relating to the target we searched. We can use this to make further queries to find data in other tables relating to this target:
62 
63 ```
64 # get a list of integers corresponding to the pk's for all entries related to the search target
65 id_list = search.get_target_id_list()
66 ```
67 
68 **2. Getting pdb data - currently apo pdb for given crystal name**
69 
70 First, init an instance of GetPdbData from fragalysis_preproc/data.py
71 
72 ```
73 from data import GetPdbData
74 
75 search = GetPdbData()
76 ```
77 
78 Next, we can get a string representation of the PDB file corresponding to the apo structure for the bound-state structure, given an input 'code', i.e. crystal name
79 
80 ```
81 # get apo pdb for bound-state of NUDT5A-x0123 in string representation
82 apo_pdb = search.get_pdb_file(code='NUDT5A-x0123')
83 ```
84 
85 **3. Getting molecule data - all data relating to molecule from molecule table**
86 
87 First, init an instance of GetMoleculesData from fragalysis_preproc/data.py
88 
89 ```
90 from data import GetMoleculesData
91 
92 search = GetMoleculesData()
93 ```
94 
95 Further searches require a target id, which can be found with the api, as shown in point 1. Alternativley, this can be done from the instance we just set up, for a given target name:
96 
97 ```
98 # get the target id's for further operations for NUDT5A
99 id_list = search.get_target_ids(target='NUDT5A')
100 ```
101 
102 The GetMoleculesData object can either operate on one object at a time, or the entire id list.
103 
104 Here, I'll show an example that will just cover the first target id in `id_list`
105 
106 To search the API for moleaules relating to the target id, we need to sett the molecule url:
107 
108 ```
109 # set the molecule url for the current instance of GetMoleculesData to the first id in the id_list from above
110 url = search.set_molecule_url(target_id=id_list[0])
111 ```
112 
113 Now, we can get the data held in the Molecules table in the fragalysis db in json format (we need to use the returned url from the above snippet. The reason why will become clear later)
114 
115 ```
116 # get a json response from the url we set above
117 results = get_molecules_json(url=url)
118 ```
119 
120 If we want to operate on all target id's related to a target, we can use a built-in function. Make sure to first set the target_ids parameter with `get_target_ids` as shown earlier.
121 
122 ```
123 # get all molecule data (json) related to all of the ids in id_list
124 results_table = search.get_all_mol_responses()
125 ```
126 
127 This returns a pandas dataframe containing: code(crystal name), pdb(apo pdb file as string), sdf(sdf of the bound ligand as string) and smiles(smiles string of the bound ligand)
128 
129 You can then work on this dataframe however you wish.
130 
131 Alternativley, there is a script which can be used to download the apo pdb's and sdf files, and a csv file containing the smiles strings for each hit for a given target, as shown in the next section
